变量赋值顺序

文件头声明后直接赋值；显示面板赋值；外部赋值。外部赋值如下。

同时外部拖拽赋值即可。

则赋值顺序经过实验，得到，声明直接赋值 -> 检视面板赋值 -> Awake -> OnEnable -> Start -> 外部赋值。

对一个对象赋值，或者准确说从外部对一个对象的属性或者成员变量赋值的前提是，该对象已经有了，所以外部赋值常常是最后的。

例外情况就是两个脚本挂载在同一Object下，则Start的顺序不可知，在Start中进行的外部赋值自然不可知。
